## 1. 启动服务

解压后双击 `bridge.exe`（Windows）或运行 `./bridge`（macOS/Linux），终端出现以下日志即启动成功：

```
time=2026-07-13 15:04:05 level=INFO msg="正在启动 Agent" id=kimi
time=2026-07-13 15:04:05 level=INFO msg="正在启动 Agent" id=opencode
time=2026-07-13 15:04:05 level=INFO msg="正在启动 Agent" id=codex
time=2026-07-13 15:04:05 level=INFO msg="正在启动 Agent" id=hermes
time=2026-07-13 15:04:05 level=INFO msg="ACP 握手成功" id=kimi protocol=1
time=2026-07-13 15:04:05 level=INFO msg="ACP 握手成功" id=opencode protocol=1
time=2026-07-13 15:04:05 level=INFO msg="ACP 握手成功" id=codex protocol=1
time=2026-07-13 15:04:05 level=INFO msg="Admin HTTP 服务启动" address=:9202
```

> 日志中可能会出现 TunnelService 连接失败的 WARN 信息，这是正常的 — 表示当前未连接 SaaS 平台，不影响测试页面的使用。

## 2. 浏览器测试

打开 `test_saas.html`（双击或在浏览器地址栏输入 `http://localhost:9202/test_saas.html`）。

**测试流程：**

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 点蓝色 **"连接"** 按钮 | 状态变为 "已连接"，Agent 下拉框自动填充 Agent 名称 |
| 2 | 选择一个 Agent（如 kimi），点 **"创建会话"** | 日志区返回 `sessionId`（如 `sess_xxx`） |
| 3 | 输入提示词（如 "1+1="），点 **"发送提示词"** | 日志区出现流式 `session/update` 消息 |
| 4 | 复制 sessionId，粘贴到"加载会话"区，点 **"加载"** | 日志显示加载成功 |
| 5 | 加载成功后，点 **"查询消息"** | 显示该会话的历史消息内容 |

## 3. 连接真实 SaaS

编辑 `%USERPROFILE%\.zleap\tunnel\config.json`：

```json
{
  "server_url": "wss://your-saas-domain.com/ws",
  "bridge_id": "bridge_xxxxx",
  "token": "ws_token_xxx",
  "admin_port": 9202
}
```

或者设置环境变量（优先级更高）：

```powershell
$env:ZLEAP_SERVER_URL = "wss://your-saas-domain.com/ws"
```

然后重启 bridge.exe 即可。

## 4. 自动化测试

项目提供了 `scripts/test_all_agents.py` 脚本，自动对所有已注册的 Agent 执行完整测试：

```powershell
# 确保 bridge 已启动，然后在另一个终端运行：
py -3.12 scripts\test_all_agents.py
```

脚本会自动完成以下测试序列：

| 步骤 | 操作 | 说明 |
|:----|:-----|:------|
| 1 | 连接 Bridge | 自动连接 `ws://localhost:9202/ws/admin` |
| 2 | 获取 Agent 列表 | 读取 Bridge 中已注册的 Agent |
| 3 | session/new | 创建新会话 |
| 4 | sessions/list | 列出所有会话 |
| 5 | session/prompt | 发送提示词并收集流式回复 |
| 6 | sessions/messages | 查询会话消息历史 |
| 7 | session/load | 加载已有会话 |
| 8 | session/load(异常) | 加载不存在的会话（错误处理） |

未安装的 Agent 会自动输出安装指引和配置提示。

## 5. 常用命令

```powershell
# 调试模式（日志更详细）
bridge.exe --debug

# 指定端口（默认 9202）
bridge.exe --port 9202

# 查看端口占用
netstat -ano | findstr :9202

# 手动停止旧进程（bridge 启动时已自动清理旧进程，通常不需要手动执行）
taskkill /IM bridge.exe /F
```

## 6. 常见问题

**Q：启动后没有 Agent？**
A：确认 `kimi`、`opencode`、`codex`、`hermes acp` 等命令在命令行中可直接运行。Bridge 会自动扫描 PATH 发现已安装的 Agent。

**Q：创建会话报错？**
A：先点"连接"，确认 WebSocket 状态为"已连接"。

**Q：端口 9202 被占用？**
A：Bridge 启动时会自动检测并清理占用 9202 端口的旧 bridge 进程。如果被非 bridge 程序占用，运行 `netstat -ano | findstr :9202` 查找 PID，然后用 `taskkill /PID 进程号 /F` 杀掉。

**Q：Hermes Agent 连接失败？**
A：确认已安装 ACP 附加组件：`pip install -e '.[acp]'`。ACP 模式需要该组件，否则 `hermes acp` 命令不可用。
